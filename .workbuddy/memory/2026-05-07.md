## 第二轮深度代码 Review（15:33）
关键发现：
1. OrchestratorAgent 的 chatStream 并非真正流式（子 Agent 同步等待再一次性推送）
2. ResumeAgent/NegotiationAgent 传了 ChatMemory.CONVERSATION_ID 但 ChatClient 未配置 Memory Advisor，多轮记忆不生效
3. YuManus 同时有 @Component 注解（Spring Bean）又在 Controller 每次 new 实例，二义性
4. JwtUtil 的 SECRET 硬编码在代码中（安全风险）
5. SessionManager 用 ConcurrentHashMap 存会话，重启后数据丢失
6. font-asian 依赖错误标记为 test scope，导致生产环境 PDF 中文渲染失败
7. OrchestratorAgent 意图识别结果无模糊匹配兜底，模型返回带标点时会路由到 GENERAL
8. doChatWithRag 的 MultiQuery 扩展只取第一个查询，其余扩展查询未真正用于召回
